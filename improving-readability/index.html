First Design Task

15th February, 2022

One of the first design tasks I was assigned for the Atlas design team was to design a breadcrumb component. The design process I followed while implementing this component consisted of two phases, discovery and design.

Breadcrumb Component Task

The component I was tasked with to design was a breadcrumb component. The breadcrumb is a compact navigation system used to show a user’s location within the site’s hierarchy.

Discovery Phase

During the discovery phase I explored the best practices for the component including its functionality and accessibility. I explored Nielson Norman Group and Designing for web accessibility W3 to note down the best practices. I used a Figma file to store my design process during the discovery phase.

I visited other design systems to see how they approach the breadcrumb component. The design systems I approached were Atlassian, Ant and Carbon. I made note of all the common patterns these design systems use for their breadcrumb component. Keeping the developers in mind, I took note of the props each design system used in their breadcrumb component because after completing my design for the breadcrumb, I need to provide developer specifications to the Atlas developers.

Design Phase

After a few hard working hours in Figma I successfully created the breadcrumb component. With all of the research I gathered, I combined them into one final design solution. The component consists of a maximum of eight breadcrumb items that are separated using the ">" symbol. Each breadcrumb item is a link except the current page breadcrumb.

To implement this component, I reused the Atlas link component and added it to the breadcrumb component along with the separators. Re-using the link component and adding it to be a piece of the breadcrumb component reminded of a great quote said by Drew Bridewell:

"A design system acts as the connective tissue that holds together your entire platform".

Designing the component in the Travelport's Figma file create what is known as a single source of truth. This means that when other designer want to use the breadcrumb component in their own wireframes they can do so without "breaking" the component. Atlas has control over the source of truth breadcrumb component and, therefore, if any design changes need to be made for the component, an Atlas designer like me can come in and edit the source of truth component. The breadcrumb component the other designers use which is can be seen as the "child" of the source of truth component (parent), will inherit any changes made to its parent. This does not cause any interuption to the designers using the component, it simply updates.
